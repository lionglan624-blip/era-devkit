# 回帰チェック — 過去の修正との差分調査

対象: $ARGUMENTS

以下を順に実行し、過去に同様の修正が行われていないか調査せよ。

## 1. 関連コミット検索

対象のキーワード（関数名、フラグ名、ファイル名など）で過去コミットを検索:

```
git log --oneline --all --grep="<keyword>" -- <file>
```

複数キーワードで試行し、関連コミットを列挙する。

## 2. 過去の修正内容確認

見つかったコミットの差分を確認:

```
git show <commit>
```

今回の修正と同じ意図の変更があるか判定する。

## 3. 上書き検出

過去の修正が後続コミットで消されていないか確認:

```
git log <past-commit>..HEAD -- <file>
```

上書きコミットが見つかったら `git show` でその差分も確認する。

## 4. セッションログ検索

`session-search.py` で過去の修正セッションを検索:

```
python src/tools/python/session-search.py "<keyword>" --after <1週間前> --type user
python src/tools/python/session-search.py "<keyword>" --after <1週間前> --type text
```

関連セッションが見つかったら `--session <id> --timeline` で調査内容を確認する。

## 5. 現在のコードとの差分比較

過去の修正と現在のコードを比較し、以下を報告:

- 過去の修正で追加されたガード/ロジックが現在も残っているか
- 残っていない場合、どのコミットで消えたか
- 今回の修正と過去の修正の差異（同一修正の再適用 / 別アプローチ / 根本原因が異なる）

## 6. 結論

| 項目 | 結果 |
|------|------|
| 過去に同様の修正があったか | Yes/No |
| 上書きされていたか | Yes/No (コミットID) |
| 今回の修正との関係 | 同一再適用 / 別アプローチ / 別原因 |
| 再発防止策 | テスト追加 / コメント追記 / etc. |
