## Implementation Notes

### File Structure

```
tools/ErbLinter/
├── Program.cs              # Add --dead-code option
├── Config/
│   ├── LinterOptions.cs    # DeadCode, EntryPointsFile options
│   └── SystemEntryPoints.txt  # Entry point definitions
└── Analyzer/
    ├── FunctionAnalyzer.cs    # Existing (reused)
    └── DeadCodeAnalyzer.cs    # Dead code detection
```

### Key Design Decisions

1. **Info Level**: DEAD001 is Info, not Warning (may have false positives)
2. **Entry Point File**: External config allows customization without recompile
3. **Regex Patterns**: Support `^PATTERN$` for flexible matching
4. **Existing Integration**: Reuse FunctionIndex and FunctionAnalyzer

### Results

- Scanned 402 ERB files with 7074 functions
- Detected 616 potential dead code issues (DEAD001)
- 22 entry point patterns defined to reduce false positives
