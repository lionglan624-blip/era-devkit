# WBS-023: Process God Object Split Phase 3

## Overview

Extract command dispatch coordination from Process.ScriptProc.cs into a testable CommandDispatcher service.

**Scope Refinement**: After analysis, `runScriptProc()` is tightly coupled with ProcessState and skip state cannot be fully extracted without significant refactoring. This phase focuses on extracting the command dispatch coordination layer, which provides:
- Testable command initialization
- Injectable command dispatch
- CommandRegistry coordination

## Design

### Interface Definition (Final)

```csharp
internal interface ICommandDispatcher
{
    void InitializeCommands();
    bool TryExecuteCommand(ScriptCommandContext ctx, InstructionLine func);
    (bool found, bool continueExec) TryExecuteFlowCommand(ScriptCommandContext ctx, InstructionLine func);
    bool HasNormalCommand(FunctionCode code);
    bool HasFlowCommand(FunctionCode code);
    int NormalCommandCount { get; }
    int FlowCommandCount { get; }
}
```

### Implementation Strategy (Final)

1. **Wrap CommandRegistry** - CommandDispatcher holds and initializes the registry
2. **Process delegates** - Process.doNormalFunction/doFlowControlFunction delegate to GlobalStatic.CommandDispatcher
3. **Skip state remains** - Skip state stays in Process (too deeply embedded to extract safely)

## Log

### Task 1: Analysis (2025-12-12)

**Command dispatch flow analysis:**

1. `runScriptProc()` loop dispatches to:
   - `doNormalFunction()` for normal commands
   - `doFlowControlFunction()` for flow control

2. Both methods use hybrid approach:
   - Try CommandRegistry first (command pattern from Feature 005)
   - Fall back to switch statement for unmigrated commands

3. Skip state flags (remain in Process):
   - `skipPrint` - Controls print skipping
   - `saveSkip` - Saved skip state for NOSKIP blocks
   - `userDefinedSkip` - User-defined skip (SKIPDISP command)

**Decision**: Skip state too deeply embedded to extract. Focus on command dispatch coordination.

### Tasks 2-5: Implementation (2025-12-12)

Created ICommandDispatcher with 7 methods:
- InitializeCommands
- TryExecuteCommand
- TryExecuteFlowCommand
- HasNormalCommand
- HasFlowCommand
- NormalCommandCount
- FlowCommandCount

Process.ScriptProc.cs now uses GlobalStatic.CommandDispatcher:
- Removed _commandRegistry field
- Removed command registration code (~80 lines)
- doNormalFunction/doFlowControlFunction delegate to dispatcher
