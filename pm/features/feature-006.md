# Feature 006: 咲夜NTR口上分割

## Status: [DONE]

## Overview

咲夜の `NTR口上.ERB`（12,774行、124関数）を論理的なシーン単位で複数ファイルに分割し、保守性と可読性を向上させる。

## Problem

現状の `Game/ERB/口上/4_咲夜/NTR口上.ERB`:
- **12,774行** の巨大ファイル（他キャラの同種ファイルより大幅に大きい）
- **124個の関数** が単一ファイルに集約
- 編集時のナビゲーションが困難
- 特定シーンの修正時に関係ない部分まで目を通す必要
- gitでの変更追跡が困難（大きなdiff）

### 比較参考（他キャラのNTR口上.ERB）
| キャラ | 行数 |
|--------|------|
| 美鈴 | ~2,000行 |
| 小悪魔 | ~3,000行 |
| パチュリー | ~2,500行 |
| **咲夜** | **12,774行** |
| レミリア | ~2,000行 |

## Goals

1. **ファイル分割**: シーン番号ごとに複数ファイルへ分離
2. **保守性向上**: 各ファイル2,000-3,000行程度に抑制
3. **命名規則統一**: 他キャラと整合性のある分割パターン
4. **動作維持**: ゲーム動作に影響を与えない

## Technical Analysis

### 現状の関数構成

```
NTR口上.ERB (124関数)
├── K4_1系 (4関数): 基本NTRシーン
├── K4_2系 (7関数): シーン2
├── K4_3系 (6関数): シーン3
├── K4_4系 (3関数): シーン4
├── K4_5系 (7関数): シーン5
├── K4_6系 (3関数): シーン6
├── K4_7系 (3関数): シーン7
├── K4_8系 (15関数): 大規模シーン8 (~2,500行)
├── K4_9系 (21関数): 最大シーン9 (~3,600行)
├── K4_11-22系 (17関数): 追加シーン群
├── K4_A共通系 (挑発等)
└── MSG系 (2関数): メッセージ
```

### 分割案

| 新ファイル名 | 内容 | 推定行数 |
|-------------|------|----------|
| `NTR口上_基本.ERB` | K4_1～K4_7 | ~3,500行 |
| `NTR口上_シーン8.ERB` | K4_8系全て | ~2,500行 |
| `NTR口上_シーン9.ERB` | K4_9系全て | ~3,600行 |
| `NTR口上_追加.ERB` | K4_11～K4_22, 共通, MSG | ~3,200行 |

### ERB分割の注意点

1. **関数参照**: ERBは関数名でグローバル参照されるため、ファイル分割しても動作に影響なし
2. **ヘッダー**: 各ファイルに適切なコメントヘッダー追加
3. **エンコーディング**: UTF-8維持
