# WBS-083: 起床後の状態注入

## Overview

Headlessテストで起床処理（`@EVENTTRAIN`）完了後に状態を注入する機能を実装。
起床処理でリセットされるキャラクター位置などを、テストに必要な状態に設定できるようにする。

## Implementation Notes

### Detection Logic

```
状態遷移:
Train_Begin (0x10) → Train_CallEventTrain (0x11) → Train_CallShowStatus (0x12)
                                                    ↑ ここで注入
```

検出条件:
1. 前の状態が `Train_CallEventTrain` (0x11)
2. 現在の状態が `Train_CallShowStatus` (0x12) または後続状態
3. 保留中の起床後注入が存在
