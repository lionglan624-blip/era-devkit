## Implementation Notes

### File Structure (Planned)

```
tools/
└── ErbLinter/
    ├── ErbLinter.csproj
    ├── Program.cs              # Entry point, CLI
    ├── Config/
    │   └── LinterConfig.cs     # Settings, ignore rules
    ├── Data/
    │   ├── CsvLoader.cs        # CSV parsing
    │   └── VariableRegistry.cs # Variable definitions
    ├── Parser/
    │   ├── Token.cs            # Token types
    │   ├── Tokenizer.cs        # Line tokenization
    │   └── FunctionIndex.cs    # Function registry
    ├── Analyzer/
    │   ├── SyntaxAnalyzer.cs   # Block matching
    │   ├── VariableAnalyzer.cs # Variable validation
    │   ├── FunctionAnalyzer.cs # Call validation
    │   └── StyleAnalyzer.cs    # Naming rules
    └── Reporter/
        ├── Issue.cs            # Issue data structure
        ├── TextReporter.cs     # Text output
        └── JsonReporter.cs     # JSON output
```

### Key Design Decisions

1. **Standalone Tool**: No uEmuera dependency, can run independently
2. **Incremental Parsing**: Line-by-line, no full AST (simplicity over completeness)
3. **Level-based Filtering**: Error/Warning/Info allows flexible usage
4. **Extensible Analyzers**: Each analyzer is independent, easy to add new checks
