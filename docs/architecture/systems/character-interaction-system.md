# キャラクター間交流システム設計

## Status: DRAFT

**Target Version**: v1.5 (S1)

## 概要

住人同士の交流を商業フルプライスゲーム級のアルゴリズムで拡張する設計。
既存の「住人同士の交流.ERB」を基盤に、多軸関係性・社会構造・情報伝播を追加。

---

## 1. 既存システム分析

### 1.1 現在の実装

**ファイル**: `住人同士の交流.ERB`

```erb
; 関係性変数
TA:キャラA:キャラB:TA_好感度  ; キャラ間好感度（2次元配列）

; 行為種別コード
300X = 会話
301X = お茶
302X = スキンシップ
310X = 尻を撫でる
311X = 抱きつく
312X = キスする
313X-316X = 愛撫系（胸/アナル/クリ/指挿れ）
350X-357X = 性交系（裸/着衣/アナル/後戯）
; Xは0=受け, 1=攻め
```

**現在のロジック**:
1. 同室時に20%確率で交流イベント発生
2. `TA_好感度 >= 体を許す好感度` で性行為解禁
3. 攻め/受けの立場判定あり

### 1.2 課題

| 課題 | 詳細 |
|------|------|
| 単軸関係性 | 好感度のみで関係を表現 |
| 行動選択が単純 | 確率＋閾値のみ |
| 社会構造なし | 派閥・グループ概念なし |
| 情報伝播なし | 噂・秘密の共有なし |
| 三角関係なし | 嫉妬の発生・伝播なし |

## 2. 多軸関係性システム

### 2.1 関係性の7軸

```erb
; ========================================
; 関係軸定義 (TA変数)
; ========================================

; TA:キャラA:キャラB:軸ID で管理
; 範囲: 0-100

#DIM CONST TA_好感度 = 0    ; 好意・親しみ（既存）
#DIM CONST TA_信頼度 = 1    ; 信頼・頼れる
#DIM CONST TA_尊敬度 = 2    ; 敬意・憧れ
#DIM CONST TA_親密度 = 3    ; 距離感の近さ
#DIM CONST TA_嫉妬度 = 4    ; 嫉妬・独占欲
#DIM CONST TA_敵意度 = 5    ; 反感・対立
#DIM CONST TA_性的関心 = 6  ; 性的な意識
```

### 2.2 軸の意味と影響

| 軸 | 意味 | 上昇条件 | 下降条件 | 行動への影響 |
|----|------|----------|----------|--------------|
| **好感度** | 好意 | 会話、贈り物 | 口論、裏切り | 基本的な交流意欲 |
| **信頼度** | 信用 | 助ける、秘密を守る | 嘘、約束破り | 相談、秘密共有 |
| **尊敬度** | 敬意 | 能力発揮、指導 | 失態、醜態 | 従う/リードする |
| **親密度** | 距離感 | 時間共有、スキンシップ | 疎遠、回避 | 物理的接触の許容 |
| **嫉妬度** | 独占欲 | 好意持つ相手の他者接近 | 時間経過、諦め | 妨害、独占行動 |
| **敵意度** | 反感 | 口論、利害対立 | 和解、謝罪 | 回避、攻撃 |
| **性的関心** | 性的意識 | 性行為、魅力 | 時間経過 | 性的行動の選択 |

## 3. 関係性ダイナミクス

```erb

; ========================================
```

```erb
      TA:A:B:TA_性的関心 = TA:A:B:TA_性的関心 * 97 / 100

      IF !CFLAG:A:今日_{B}_接触
        TA:A:B:TA_親密度 = TA:A:B:TA_親密度 * 99 / 100
      ENDIF
    NEXT
  NEXT
  RETURN
```

## 4. 嫉妬システム

### 4.1 嫉妬の発生

```erb
; ========================================
; 嫉妬システム
; ========================================

@JEALOUSY_CHECK_ALL(キャラA, キャラB)
  ; キャラAとキャラBの親密な行動を見て、
  ; 両者に好意を持つ第三者が嫉妬

  FOR C, 0, CHARANUM - 1
    SIF C == キャラA || C == キャラB
      CONTINUE
    SIF CFLAG:C:現在位置 != CFLAG:キャラA:現在位置
      CONTINUE  ; 同室でないと目撃しない

    CALL JEALOUSY_CHECK(C, キャラA, キャラB)
  NEXT
  RETURN

@JEALOUSY_CHECK(観察者, 行為者A, 行為者B)
  ; 観察者が行為者の親密な行動を目撃

  LOCAL 好意A = TA:観察者:行為者A:TA_好感度
  LOCAL 好意B = TA:観察者:行為者B:TA_好感度

  ; 行為者Aに好意を持っている場合
  IF 好意A > 40
    LOCAL 嫉妬量 = (好意A - 30) / 3 + RAND:15
    TA:観察者:行為者B:TA_嫉妬度 += 嫉妬量
    TA:観察者:行為者B:TA_敵意度 += 嫉妬量 / 3

    ; 観察者の感情更新
    CALL EMOTION_UPDATE(観察者, "嫉妬目撃", 行為者A)
  ENDIF

  ; 行為者Bに好意を持っている場合
  IF 好意B > 40
    LOCAL 嫉妬量 = (好意B - 30) / 3 + RAND:15
    TA:観察者:行為者A:TA_嫉妬度 += 嫉妬量
    TA:観察者:行為者A:TA_敵意度 += 嫉妬量 / 3
  ENDIF

  RETURN
```

### 4.2 嫉妬による行動変化

```erb
@JEALOUSY_BEHAVIOR_MODIFIER(キャラA, キャラB)
  ; 嫉妬度が高いと行動が変化
  LOCAL 嫉妬 = TA:キャラA:キャラB:TA_嫉妬度

  IF 嫉妬 > 70

  ELSEIF 嫉妬 > 50
    ; 中程度: 嫌味、皮肉
    RESULTS = "皮肉"
  ELSEIF 嫉妬 > 30
    ; 軽度: 距離を置く
    RESULTS = "回避"
  ELSE
    RESULTS = "通常"
  ENDIF
  RETURN
```

## 5. 派閥システム

### 5.1 派閥定義

```erb
; ========================================
; 派閥システム
; ========================================

; 派閥ID
#DIM CONST 派閥_主人派 = 1     ; 主人に忠実
#DIM CONST 派閥_訪問者派 = 2   ; NTR進行で発生
#DIM CONST 派閥_中立派 = 0     ; どちらでもない

@GET_FACTION(キャラ)
  ; キャラの現在の派閥を動的に判定
  LOCAL 主人寄り = CFLAG:キャラ:好感度
  LOCAL 訪問者寄り = CFLAG:キャラ:屈服度

  ; NTR進行度も考慮
  IF TALENT:キャラ:NTR
    訪問者寄り += 2000
  ENDIF
  IF TALENT:キャラ:浮気公認 >= 2
    訪問者寄り += 1000
  ENDIF

  ; 判定
  IF 主人寄り > 訪問者寄り * 2
    RESULT = 派閥_主人派
  ELSEIF 訪問者寄り > 主人寄り
    RESULT = 派閥_訪問者派
  ELSE
    RESULT = 派閥_中立派
  ENDIF
  RETURN

@GET_FACTION_NAME(派閥ID)
  SELECTCASE 派閥ID
    CASE 派閥_主人派
      RESULTS = "主人派"
    CASE 派閥_訪問者派
      RESULTS = "訪問者派"
    CASEELSE
      RESULTS = "中立派"
  ENDSELECT
  RETURN
```

### 5.2 派閥間関係

```erb
@FACTION_RELATION_MODIFIER(キャラA, キャラB)
  ; 派閥関係による交流補正
  LOCAL 派閥A = CALL GET_FACTION(キャラA)
  LOCAL 派閥B = CALL GET_FACTION(キャラB)

  IF 派閥A == 派閥B
    ; 同派閥: 協力ボーナス
    RESULT = 20  ; 交流スコア+20
  ELSEIF 派閥A == 派閥_主人派 && 派閥B == 派閥_訪問者派
    ; 対立派閥
    RESULT = -30  ; 交流スコア-30
  ELSEIF 派閥A == 派閥_訪問者派 && 派閥B == 派閥_主人派
    RESULT = -30
  ELSE
    ; 中立が絡む場合
    RESULT = 0
  ENDIF
  RETURN

@FACTION_SPECIAL_EVENT(キャラA, キャラB)
  ; 派閥間の特殊イベント
  LOCAL 派閥A = CALL GET_FACTION(キャラA)
  LOCAL 派閥B = CALL GET_FACTION(キャラB)

  ; 主人派が訪問者派を説得しようとする
  IF 派閥A == 派閥_主人派 && 派閥B == 派閥_訪問者派
    IF TA:キャラA:キャラB:TA_好感度 > 50 && RAND:100 < 20
      RESULTS = "説得イベント"
      RETURN 1
    ENDIF
  ENDIF

  ; 訪問者派同士の秘密共有
  IF 派閥A == 派閥_訪問者派 && 派閥B == 派閥_訪問者派
    IF RAND:100 < 30
      RESULTS = "共犯意識イベント"
      RETURN 1
    ENDIF
  ENDIF

  RETURN 0
```

## 6. 噂システム

### 6.1 噂の生成と伝播

```erb
; ========================================
; 噂システム
; ========================================

; 噂種別
; "浮気"     - キャラXが訪問者と関係
; "密会"     - キャラXとキャラYが密会
; "妊娠"     - キャラXが妊娠
; "不仲"     - キャラXとキャラYが不仲
; "秘密"     - キャラXの秘密

@RUMOR_CREATE(発生源, 噂種別, 対象, 詳細)
  ; 新しい噂を生成
  LOCAL 噂ID = FLAG:次の噂ID
  FLAG:次の噂ID += 1

  ; 噂データ保存
  FLAGS:噂_{噂ID}_種別 = 噂種別
  FLAG:噂_{噂ID}_対象 = 対象
  FLAGS:噂_{噂ID}_詳細 = 詳細
  FLAG:噂_{噂ID}_発生源 = 発生源
  FLAG:噂_{噂ID}_信憑性 = 100  ; 初期は確実
  FLAG:噂_{噂ID}_拡散度 = 1    ; 知っている人数

  ; 発生源は噂を知っている
  SETBIT FLAG:噂_{噂ID}_知者, 発生源

  RESULT = 噂ID
  RETURN

@RUMOR_SPREAD(噂ID)
  ; 噂が館内に広がる（日次処理）

  FOR キャラ, 0, CHARANUM - 1
    ; 既に知っている場合はスキップ
    SIF GETBIT(FLAG:噂_{噂ID}_知者, キャラ)
      CONTINUE

    ; 伝播判定: 知っている人との親密度
    FOR 知者, 0, CHARANUM - 1
      SIF !GETBIT(FLAG:噂_{噂ID}_知者, 知者)
        CONTINUE
      SIF CFLAG:知者:現在位置 != CFLAG:キャラ:現在位置
        CONTINUE  ; 同室でないと伝わらない

      ; 伝播確率 = 親密度/2 + 噂好き補正
      LOCAL 確率 = TA:キャラ:知者:TA_親密度 / 2
      IF TALENT:知者:おしゃべり
        確率 += 20
      ENDIF

      IF RAND:100 < 確率
        SETBIT FLAG:噂_{噂ID}_知者, キャラ
        FLAG:噂_{噂ID}_拡散度 += 1
        ; 信憑性は伝播で低下
        FLAG:噂_{噂ID}_信憑性 = FLAG:噂_{噂ID}_信憑性 * 90 / 100
        CALL RUMOR_RECEIVE(キャラ, 噂ID)
        BREAK
      ENDIF
    NEXT
  NEXT
  RETURN
```

```erb
      IF TA:キャラ:対象:TA_好感度 > 50

      IF TALENT:キャラ:世話好き
```

## 7. 交流行動のUtility AI

### 7.1 行動選択

```erb
; ========================================
; 交流行動のUtility AI
; ========================================

@INHABITANT_SELECT_ACTION_UTILITY(キャラA, キャラB)
  ; 多軸関係性からスコア計算して行動選択

  LOCAL:STR 行動候補:10
  LOCAL スコア:10

  ; 行動候補
  行動候補:0 = "会話"
  行動候補:1 = "お茶"
  行動候補:2 = "相談"
  行動候補:3 = "スキンシップ"
  行動候補:4 = "からかう"
  行動候補:5 = "愛撫"
  行動候補:6 = "性交"
  行動候補:7 = "回避"
  行動候補:8 = "説得"
  行動候補:9 = "共犯話"

  ; 関係性取得
  LOCAL 好感 = TA:キャラA:キャラB:TA_好感度
  LOCAL 信頼 = TA:キャラA:キャラB:TA_信頼度
  LOCAL 尊敬 = TA:キャラA:キャラB:TA_尊敬度
  LOCAL 親密 = TA:キャラA:キャラB:TA_親密度
  LOCAL 嫉妬 = TA:キャラA:キャラB:TA_嫉妬度
  LOCAL 敵意 = TA:キャラA:キャラB:TA_敵意度
  LOCAL 性的 = TA:キャラA:キャラB:TA_性的関心

  ; 派閥補正
  LOCAL 派閥補正 = CALL FACTION_RELATION_MODIFIER(キャラA, キャラB)

  ; スコア計算
  スコア:0 = 50 + 好感 / 5 + 派閥補正                    ; 会話
  スコア:1 = 30 + 親密 / 3 + 好感 / 10                   ; お茶
  スコア:2 = 信頼 / 2 + 尊敬 / 5                         ; 相談
  スコア:3 = 親密 / 2 + 好感 / 10                        ; スキンシップ
  スコア:4 = 20 + 敵意 / 3 - 尊敬 / 5                    ; からかう
  スコア:5 = 性的 / 2 + 親密 / 5                         ; 愛撫
  スコア:6 = 性的 / 2 + 親密 / 3 + 好感 / 10             ; 性交
  スコア:7 = 敵意 / 2 + 嫉妬 / 3 - 好感 / 5              ; 回避
  スコア:8 = 0  ; 説得（派閥条件で有効化）
  スコア:9 = 0  ; 共犯話（派閥条件で有効化）

  ; 性行為は閾値チェック（既存互換）
  IF TA:キャラB:キャラA:TA_好感度 < CFLAG:キャラB:体を許す好感度
    スコア:5 = 0
    スコア:6 = 0
  ENDIF

  ; 派閥特殊行動
  LOCAL 派閥A = CALL GET_FACTION(キャラA)
  LOCAL 派閥B = CALL GET_FACTION(キャラB)

  IF 派閥A == 派閥_主人派 && 派閥B == 派閥_訪問者派
    スコア:8 = 60 + 好感 / 5  ; 説得
  ENDIF

  IF 派閥A == 派閥_訪問者派 && 派閥B == 派閥_訪問者派
    スコア:9 = 50 + 親密 / 3  ; 共犯話
  ENDIF

  ; 嫉妬が高いと回避優先
  IF 嫉妬 > 50
    スコア:7 += 30
  ENDIF

  ; 敵意が高いと交流スコア減少
  IF 敵意 > 30
    FOR I, 0, 6
      スコア:I -= 敵意 / 3
    NEXT
  ENDIF

  ; 最高スコア選択
  LOCAL 最高 = 0
  LOCAL 選択 = 0
  FOR I, 0, 9
    IF スコア:I > 最高
      最高 = スコア:I
      選択 = I
    ENDIF
  NEXT

  RESULTS = 行動候補:選択
  RETURN
```
