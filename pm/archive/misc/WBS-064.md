## Technical Notes

### Design Decisions

1. **Process isolation** - Each worker is a separate `dotnet run` process
2. **JSON output** - Workers use `--output-mode json` for machine-readable results
3. **Memory-based scaling** - `--parallel` (no N) calculates workers from available memory
4. **Backward compatible** - Existing `--parallel N` syntax still works

### Implementation Approach

```
ProcessLevelParallelRunner
├── CalculateOptimalWorkers() - Memory-based worker count (200MB per worker)
├── Run(tests) - Parallel.ForEach with process spawning
├── RunTestInProcess(test) - Spawn worker, parse JSON result
└── ParseJsonResult(output) - Extract result from summary JSON
```

### CLI Changes

| Mode | Command | Behavior |
|------|---------|----------|
| Process (new) | `--parallel [N]` | Process-level isolation |
| Auto workers | `--parallel` | Calculate from memory |
| Fixed workers | `--parallel 4` | Use 4 processes |
| Verbose | `--verbose, -v` | Show memory usage |
