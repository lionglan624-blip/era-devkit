# Feature 047: Quick Start to Shop

## Status: [DONE]

## Summary

ニューゲーム直後にデフォルト設定でSHOP画面まで直行するクイックスタート選択肢を追加する。

## Background

現在のニューゲームでは約20以上の設定画面を経由する必要があり、開発・テスト時に時間がかかる。

### 現在のニューゲームフロー (約22ステップ)

| # | 画面 | デフォルト入力 |
|---|------|----------------|
| 1 | タイトル | 0 (最初から) |
| 2 | モード選択 (ふたなり等) | 0 (START) |
| 3 | なりきり選択 | 0 (いいや) |
| 4 | NAME入力 | 0 (変更しない) |
| 5 | CALLNAME入力 | 0 (変更しない) |
| 6 | 主人公カスタム | 1000 (決定) |

| 9 | 住人カスタム確認 | 0 (はい) |

| 11 | 処女選択 | 1 (デフォルト) |
| 12 | *(PRINTW待ち)* | - |

| 14 | *(PRINTW待ち)* | - |

## Acceptance Criteria

- [x] モード選択画面に `[9]クイックスタート` 選択肢を追加
- [x] 選択時、上記2-21の設定画面をスキップしデフォルト値を適用
- [x] `SETBIT FLAG:ＮＴＲパッチ設定,0` を事前にセットし、起床直後のNTRオプション画面もスキップ
- [x] SHOP画面に直接遷移

## Implementation Notes

### 主な変更ファイル

- `Game/ERB/SYSTEM.ERB` - `@EVENTFIRST` にクイックスタート分岐を追加

### スキップ処理

1. モード選択で `[9]` を選択した場合、以下をスキップ:
   - `CUSTOM_CHARAMAKE` (主人公カスタム)
   - `CHARA_CUSTUM` (住人カスタム)
   - `CLOTHES_CUSTOM` (服装選択)
   - `VIRGIN_CUSTOM` (処女選択)
   - `SHORT_CUT_MODE` (ショートカットモード)
   - 日付初期設定
   - 寝間着設定
   - 体の詳細設定
   - 服装変更

2. デフォルト値の適用:
   - 主人公: デフォルト名「あなた」、男性、デフォルトステータス
   - 住人: CSV初期値のまま
   - 日付: 春季1日

3. NTRオプションスキップ:
   - `SETBIT FLAG:ＮＴＲパッチ設定,0` を設定
