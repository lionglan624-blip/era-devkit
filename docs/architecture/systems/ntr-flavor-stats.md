# NTRフレーバーステータス設計提案

## Status: DRAFT

**Target Version**: v2.0, v2.3 (S2)

## 概要

NTRの説得力を高めるためのステータス拡張。「なぜ寝取られるのか」を数値とテキストで表現する。

## 背景・動機

### 現状の課題

1. **訪問者の魅力が不明確**: なぜ住人が訪問者に惹かれるのか説明がない
2. **プレイヤーとの比較がない**: 「主人より優れている」という演出ができない
3. **堕落の可視化が弱い**: 経験値は累積するが、ランクとして見えない

### 参考: era系ゲームの一般的なステータス

[eraシリーズとは - eraシリーズを語るスレ まとめWiki V3](https://seesaawiki.jp/eraseries/d/era%A5%B7%A5%EA%A1%BC%A5%BA%A4%C8%A4%CF)

- 陥落素質: 恋慕、淫乱、服従
- 感度系: 各部位ごとの感度
- 経験カウンタ: 行為ごとの累積回数

## 1. ランクシステム導入

### 基本方針

**既存の-2～+2を維持しつつ、表示時にランク変換**

```erb
; ランク定義
#DIM CONST ランク_E = -2
#DIM CONST ランク_D = -1
#DIM CONST ランク_C = 0
#DIM CONST ランク_B = 1
#DIM CONST ランク_A = 2
#DIM CONST ランク_S = 3   ; 特殊（訪問者専用など）

; 表示用関数
@GET_RANK_STR(VALUE)
  SELECTCASE VALUE
    CASE -2
      RESULTS = "E"
    CASE -1
      RESULTS = "D"
    CASE 0
      RESULTS = "C"
    CASE 1
      RESULTS = "B"
    CASE 2
      RESULTS = "A"
    CASE 3
      RESULTS = "S"
  ENDSELECT
  RETURN
```

## 2. 男性ステータス（訪問者/プレイヤー）

### 2.1 チンポステータス（細分化）

| ステータス | 範囲 | 単位 | 影響する絶頂 | 説明 |
|-----------|:----:|:----:|:------------:|------|
| **長さ** | E～S | cm | V, A | 奥への到達度 |
| **太さ** | E～S | cm | V, A | 壁への圧迫感 |
| **硬度** | E～S | - | V, A, C | 刺激の明確さ |
| **持続力** | E～S | - | 全部位 | 射精までの時間（精力消費効率） |

#### cm値の保存と成長

```erb
; 変数定義
; CFLAG:チンポ_長さ_ランク = -2～+3 (計算用)
; CFLAG:チンポ_長さ_mm = 実測値 (mm単位で保存、成長用)

@INIT_CHINPO_SIZE(CHARA, ランク)
  ; ランクからcm範囲を決定し、その範囲内でランダム値を設定
  SELECTCASE ランク
    CASE -2  ; E
      CFLAG:CHARA:チンポ_長さ_mm = 60 + RAND:21  ; 60-80mm
    CASE -1  ; D
      CFLAG:CHARA:チンポ_長さ_mm = 80 + RAND:21  ; 80-100mm
    CASE 0   ; C
      CFLAG:CHARA:チンポ_長さ_mm = 100 + RAND:31 ; 100-130mm
    CASE 1   ; B
      CFLAG:CHARA:チンポ_長さ_mm = 130 + RAND:31 ; 130-160mm
    CASE 2   ; A
      CFLAG:CHARA:チンポ_長さ_mm = 160 + RAND:31 ; 160-190mm
    CASE 3   ; S
      CFLAG:CHARA:チンポ_長さ_mm = 190 + RAND:41 ; 190-230mm
  ENDSELECT
  CFLAG:CHARA:チンポ_長さ_ランク = ランク
  RETURN

@GROW_CHINPO(CHARA, 成長mm)
  ; mm単位で成長（1mm～数mm）
  CFLAG:CHARA:チンポ_長さ_mm += 成長mm

  ; ランクの再計算
  LOCAL mm = CFLAG:CHARA:チンポ_長さ_mm
  IF mm >= 190
    CFLAG:CHARA:チンポ_長さ_ランク = 3  ; S
  ELSEIF mm >= 160
    CFLAG:CHARA:チンポ_長さ_ランク = 2  ; A
  ELSEIF mm >= 130
    CFLAG:CHARA:チンポ_長さ_ランク = 1  ; B
  ELSEIF mm >= 100
    CFLAG:CHARA:チンポ_長さ_ランク = 0  ; C
  ELSEIF mm >= 80
    CFLAG:CHARA:チンポ_長さ_ランク = -1 ; D
  ELSE
    CFLAG:CHARA:チンポ_長さ_ランク = -2 ; E
  ENDIF
  RETURN

@GET_CHINPO_CM_STR(CHARA)
  ; 表示用: "Bランク (14.2cm)"
  LOCAL mm = CFLAG:CHARA:チンポ_長さ_mm
  LOCAL ランク文字列 = GET_RANK_STR(CFLAG:CHARA:チンポ_長さ_ランク)
  RESULTS = ランク文字列 + "ランク (" + TOSTR(mm/10) + "." + TOSTR(mm%10) + "cm)"
  RETURN
```

**表示例**:
```
あなたのチンポ: Dランク (9.2cm)
訪問者のチンポ: Aランク (17.5cm)
```

#### 総合ランク計算

```erb
@CALC_CHINPO_RANK(CHARA)
  LOCAL 合計 = 0
  合計 += CFLAG:CHARA:チンポ_長さ_ランク
  合計 += CFLAG:CHARA:チンポ_太さ_ランク
  合計 += CFLAG:CHARA:チンポ_硬度
  合計 += CFLAG:CHARA:チンポ_持続力
  合計 += CFLAG:CHARA:チンポ_精液量
  合計 += CFLAG:CHARA:チンポ_回復力

  ; 平均を四捨五入 (6項目)
  RESULT = 合計 / 6
  RETURN
```

#### 絶頂計算への影響

```erb
@CALC_ORGASM_MODIFIER(行為者, 部位)
  ; 部位別にチンポステータスの影響を計算
  LOCAL 修正 = 100  ; 基準100%

  SELECTCASE 部位
    CASE "V"  ; 膣絶頂
      ; 長さ: 奥への到達 (ポルチオ刺激)
      修正 += CFLAG:行為者:チンポ_長さ_ランク * 10
      ; 太さ: 壁への圧迫
      修正 += CFLAG:行為者:チンポ_太さ_ランク * 8
      ; 硬度: 刺激の明確さ
      修正 += CFLAG:行為者:チンポ_硬度 * 6

    CASE "A"  ; アナル絶頂
      ; 太さが特に重要
      修正 += CFLAG:行為者:チンポ_太さ_ランク * 12
      修正 += CFLAG:行為者:チンポ_長さ_ランク * 6
      修正 += CFLAG:行為者:チンポ_硬度 * 6

    CASE "C"  ; クリトリス絶頂
      ; 硬度のみ影響（直接刺激）
      修正 += CFLAG:行為者:チンポ_硬度 * 4

    CASE "B"  ; 胸絶頂
      ; チンポは影響しない
      修正 = 100
  ENDSELECT

  ; 持続力: 全部位共通で快感蓄積に影響
  IF 部位 != "B"
    修正 += CFLAG:行為者:チンポ_持続力 * 5
  ENDIF

  RESULT = 修正
  RETURN
```

### 2.2 顔ステータス（細分化）

| ステータス | 範囲 | 説明 |
|-----------|:----:|------|
| **顔立ち** | E～S | イケメン度 |
| **フェロモン** | E～S | 性的魅力 |
| **威圧感** | E～S | 支配的オーラ |

### 2.3 デフォルト値

| キャラ | 長さ | 太さ | 硬度 | 持続 | 精液 | 回復 | 顔 | 総合 |
|--------|:----:|:----:|:----:|:----:|:----:|:----:|:--:|:----:|
| **プレイヤー** | **D** | **D** | D | D | C | C | C | **D** |
| **訪問者（通常）** | B | B | A | B | A | B | B | B |
| **訪問者（強）** | A～S | A | S | A | S | A | A | A |

→ **プレイヤーはDランクスタート**（劣位から成長）
→ 訪問者は固定または訪問者タイプで変動

### 2.4 成長システム概要

| 成長方法 | 効果 | 上限 |
|----------|------|:----:|
| 特定アイテム | +1～5mm | Sランク |
| 特定イベント | +3～10mm | Sランク |
| 訓練コマンド | 持続力/硬度 +0.1 | A |

**設計意図**: 主人公は努力で訪問者に追いつける。ただしSランクへの到達は困難

### 2.5 精力 (BASE) との関係

**既存の精力システムとの統合**

| 変数 | 種類 | 性質 | 説明 |
|------|------|------|------|
| **精力** | BASE | リソース | 行為中に消費・回復する「燃料」 |
| **持続力** | CFLAG | 特性 | 精力の消費効率を決める「燃費」 |
| **回復力** | CFLAG | 特性 | 精力の回復速度を決める |

```erb
; 持続力による精力消費軽減
@CALC_STAMINA_COST(行為者, 基本消費)
  ; 持続力が高いほど消費が少ない
  ; E(-2)=140%, D(-1)=120%, C(0)=100%, B(1)=80%, A(2)=60%, S(3)=50%
  LOCAL 係数 = 100 - CFLAG:行為者:チンポ_持続力 * 20
  係数 = MAX(係数, 50)  ; 最低50%
  RESULT = 基本消費 * 係数 / 100
  RETURN

; 回復力による精力回復増加
@CALC_STAMINA_RECOVERY(行為者, 基本回復)
  ; 回復力が高いほど回復が早い
  LOCAL 係数 = 100 + CFLAG:行為者:チンポ_回復力 * 20
  RESULT = 基本回復 * 係数 / 100
  RETURN

; ========================================
; 精液濃度のハイブリッドシステム (C案)
; ========================================
; 基礎濃度（成長で上昇） × 精力係数（現在の精力で変動）

@CALC_SEMEN_CONCENTRATION(射精者)
  ; 基礎濃度: CFLAG で管理 (成長可能)
  LOCAL 基礎 = CFLAG:射精者:チンポ_精液濃度  ; -2～+3

  ; 精力率: 現在の精力 / 最大精力
  LOCAL 精力率 = BASE:射精者:精力 * 100 / MAXBASE:射精者:精力

  ; 精力係数: 精力が減ると濃度も下がる
  ;   100% → ×1.2 (満タン時ボーナス)
  ;   80%  → ×1.0 (通常)
  ;   50%  → ×0.8
  ;   20%  → ×0.6
  ;   0%   → ×0.4 (枯渇時)
  LOCAL 精力係数
  IF 精力率 >= 100
    精力係数 = 120
  ELSEIF 精力率 >= 80
    精力係数 = 100
  ELSEIF 精力率 >= 50
    精力係数 = 80
  ELSEIF 精力率 >= 20
    精力係数 = 60
  ELSE
    精力係数 = 40
  ENDIF

  ; 最終濃度 = 基礎濃度 × 精力係数 / 100
  ; 結果は -3 ～ +4 程度の範囲になりうる（表示時に丸める）
  RESULT = 基礎 * 精力係数 / 100
  RETURN

@GET_SEMEN_CONCENTRATION_DESC(射精者)
  ; 表示用の濃度説明
  LOCAL 濃度 = CALL CALC_SEMEN_CONCENTRATION(射精者)

  IF 濃度 >= 3
    RESULTS = "極濃・精臭が強い"
  ELSEIF 濃度 >= 2
    RESULTS = "濃い・ドロドロ"
  ELSEIF 濃度 >= 1
    RESULTS = "やや濃い"
  ELSEIF 濃度 >= 0
    RESULTS = "普通"
  ELSEIF 濃度 >= -1
    RESULTS = "やや薄め"
  ELSE
    RESULTS = "薄い・水っぽい"
  ENDIF
  RETURN
```

**ハイブリッドシステムのゲームプレイ効果**:

| 状況 | 基礎濃度 | 精力率 | 係数 | 実効濃度 | 演出 |
|------|:--------:|:------:|:----:|:--------:|------|
| 訪問者A初回 | A(+2) | 100% | 1.2 | +2.4→A | 濃い・ドロドロ |
| 訪問者A連続3回目 | A(+2) | 40% | 0.7 | +1.4→B | やや濃い |
| 訪問者A枯渇時 | A(+2) | 10% | 0.4 | +0.8→C | 普通 |
| 主人D初回 | D(-1) | 100% | 1.2 | -1.2→D | やや薄め |
| 主人D成長後 | B(+1) | 100% | 1.2 | +1.2→B | やや濃い |

**設計意図**:
- 濃度の**基礎値は成長で上昇**（トレーニングやアイテム）
- **連続射精で薄くなる**（精力依存）→ リアリティ
- **回復すれば元の濃度に戻る**（持続力・回復力の価値）
- **訪問者との差が動的**（初回は差が大きく、連戦で縮まる）

```erb
; 口上での活用例
@KOJO_EJACULATION_CONCENTRATION(住人, 射精者)
  LOCAL 濃度 = CALL CALC_SEMEN_CONCENTRATION(射精者)
  LOCAL 精力率 = BASE:射精者:精力 * 100 / MAXBASE:射精者:精力

  ; 濃度による基本表現
  IF 濃度 >= 2
    PRINTL （濃い…ドロドロして…匂いも強い…）
  ELSEIF 濃度 >= 1
    PRINTL （少し濃い…粘ついてる…）
  ELSEIF 濃度 >= 0
    PRINTL （普通…かな…）
  ELSE
    PRINTL （薄い…水っぽい…）
  ENDIF

  ; 精力低下による追加表現
  IF 精力率 < 50
    PRINTL （さっきより薄くなってる…疲れてきたのかな…）
  ENDIF
  RETURN
```

## 10. 刻印システムとの統合

### 10.1 刻印の永続性原則

**刻印は減らない。刻まれたものだから。**

| 刻印名 | ID | 性質 |
|--------|:--:|------|
| 苦痛刻印 | 0 | 永続 |
| 快楽刻印 | 1 | 永続 |
| 屈服刻印 | 2 | 永続 |
| 浮気快楽刻印 | 5 | 永続 |
| 浮気屈服刻印 | 6 | 永続 |

刻印は一度刻まれたら消えない。これはゲームシステムの根本原則である。

### 10.2 現行の刻印計算式（参照用）

**快楽刻印** (`TRACHECK_ORGASM.ERB`):
```erb
; 絶頂強度から快楽刻印レベルを計算
今回快楽刻印 = Ｃ絶頂強度 + Ｖ絶頂強度 * 2 + Ａ絶頂強度 * 2 + Ｂ絶頂強度
```

**浮気快楽刻印** (`NTR_UTIL.ERB`):
```erb
; 現在の刻印より高い場合のみ更新（+1ずつ上昇）
IF MARK:奴隷:浮気快楽刻印 < 浮気快楽強度
    MARK:奴隷:浮気快楽刻印 = MIN(浮気快楽強度, MARK:奴隷:浮気快楽刻印 + 1)
ENDIF
```

### 10.3 問題点: 浮気快楽刻印の際限ない上昇

現行システムでは訪問者との行為を重ねるたびに浮気快楽刻印が上昇し続ける。
刻印は減らせないため、別のアプローチで制限する必要がある。

### 10.4 解決策: 刻印スケール統一 + チンポ係数

**Option A採用: 両方の刻印を0-12スケールに統一**

| 刻印 | 旧上限 | 新上限 |
|------|:------:|:------:|
| 快楽刻印 | 3 | **12** |
| 浮気快楽刻印 | 12 | 12 |

**チンポ係数による自然な制限**

上限テーブルではなく、**刻印計算時にチンポ係数を乗算**することで自然に制限。

```erb
@CALC_CHINPO_COEFFICIENT(行為者)
  ; チンポ総合ランクから係数を算出
  ; E(-2)=0.6, D(-1)=0.8, C(0)=1.0, B(1)=1.2, A(2)=1.4, S(3)=1.6
  LOCAL 総合 = CALC_CHINPO_RANK(行為者)
  RESULT = (100 + 総合 * 20) / 100
  RETURN
```

**刻印計算への適用**:

```erb
; 快楽刻印（主人による）- TRACHECK_ORGASM.ERB
LOCAL 絶頂強度合計 = Ｃ絶頂強度 + Ｖ絶頂強度 * 2 + Ａ絶頂強度 * 2 + Ｂ絶頂強度
LOCAL チンポ係数 = CALC_CHINPO_COEFFICIENT(調教者)
今回快楽刻印 = MIN(絶頂強度合計 * チンポ係数, 12)  ; 上限12

; 浮気快楽刻印（訪問者による）- NTR_UTIL.ERB
LOCAL チンポ係数 = CALC_CHINPO_COEFFICIENT(訪問者)
浮気快楽強度 = MIN(絶頂強度合計 * チンポ係数, 12)  ; 上限12
```

**効果**:
| 状況 | 絶頂強度 | 係数 | 結果刻印 |
|------|:--------:|:----:|:--------:|
| 主人D rank | 8 | 0.8 | 快楽刻印 6 |
| 訪問者A rank | 8 | 1.4 | 浮気快楽刻印 11 |
| 主人S rank (成長後) | 8 | 1.6 | 快楽刻印 12 |

**設計意図**:
- 刻印は減らない（永続性維持）
- 同じ絶頂でも、チンポが優れているほど深く刻まれる
- 主人が成長すれば快楽刻印で浮気快楽刻印を上回れる
- 上限テーブル不要（係数で自然に制限）

### 10.5 絶頂計算へのチンポステータス統合

**部位別にチンポステータスの影響を反映**

```erb
@CALC_ORGASM_MODIFIER(行為者, 部位)
  LOCAL 修正 = 100  ; 基準100%

  SELECTCASE 部位
    CASE "V"  ; 膣絶頂
      修正 += CFLAG:行為者:チンポ_長さ_ランク * 10  ; 奥への到達
      修正 += CFLAG:行為者:チンポ_太さ_ランク * 8   ; 壁への圧迫
      修正 += CFLAG:行為者:チンポ_硬度 * 6          ; 刺激の明確さ

    CASE "A"  ; アナル絶頂
      修正 += CFLAG:行為者:チンポ_太さ_ランク * 12  ; 太さが重要
      修正 += CFLAG:行為者:チンポ_長さ_ランク * 6
      修正 += CFLAG:行為者:チンポ_硬度 * 6

    CASE "C"  ; クリトリス絶頂
      修正 += CFLAG:行為者:チンポ_硬度 * 4          ; 直接刺激

    CASE "B"  ; 胸絶頂
      修正 = 100  ; チンポ影響なし
  ENDSELECT

  ; 持続力: 全部位共通
  IF 部位 != "B"
    修正 += CFLAG:行為者:チンポ_持続力 * 5
  ENDIF

  RESULT = 修正
  RETURN
```

**適用例** (`TRACHECK_ORGASM.ERB`への統合):
```erb
; V絶頂判定時
LOCAL V修正 = CALC_ORGASM_MODIFIER(調教者, "V")
V絶頂閾値 = 基本閾値 * 100 / V修正  ; 修正が高いほど閾値が下がる

; 結果:
; - チンポステータスが高い → 絶頂しやすい → 絶頂強度が上がりやすい
; - 絶頂強度 × チンポ係数 → 刻印値
```

### 10.6 設計まとめ

| 要素 | 方針 |
|------|------|
| 快楽刻印 | 絶頂強度 × チンポ係数（上限12） |
| 浮気快楽刻印 | 絶頂強度 × チンポ係数（上限12） |
| 刻印の永続性 | 減少なし（不変） |
| チンポステータス | 絶頂しやすさ + 刻印深さに影響 |

**整合性**:
- 刻印は永続的に刻まれるもの
- チンポ係数が刻む深さを決定
- 両刻印が同じスケール（0-12）で直接比較可能
- 主人が成長すれば逆転可能
