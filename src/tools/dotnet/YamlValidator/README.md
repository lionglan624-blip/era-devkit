# YamlValidator

YAML schema validation tool for ERA game dialogue files.

## Purpose

YamlValidator validates YAML-based character dialogue files against a JSON Schema to detect structural errors, type mismatches, and missing required properties before runtime. This tool enables:

- **Pre-commit validation** - Catch YAML errors before they reach version control
- **CI/CD integration** - Automated validation in build pipelines
- **Batch validation** - Validate entire directories of dialogue files
- **Clear error reporting** - Identify specific validation failures with file and path context

The validator uses JSON Schema generated by [YamlSchemaGen](../YamlSchemaGen/README.md) to verify YAML dialogue file conformance.

## Usage Examples

### Single File Validation

Validate a single YAML dialogue file:

```bash
dotnet run --project tools/YamlValidator/ -- \
  --schema tools/YamlSchemaGen/dialogue-schema.json \
  --yaml Game/YAML/Kojo/COM_K1_0.yaml
```

**Output on success:**
```
PASS: COM_K1_0.yaml is valid
```

**Output on failure:**
```
FAIL: COM_K1_0.yaml
Error at branches[0].condition: Required property 'TALENT' is missing
```

### Directory Validation

Validate all YAML files in a directory recursively (CI mode):

```bash
dotnet run --project tools/YamlValidator/ -- \
  --schema tools/YamlSchemaGen/dialogue-schema.json \
  --validate-all Game/YAML/Kojo/
```

**Output:**
```
Validating 45 YAML files...
PASS: COM_K1_0.yaml is valid
PASS: COM_K2_0.yaml is valid
FAIL: COM_K3_0.yaml
Error at branches[1].lines: Array must contain at least 1 item

=== Validation Summary ===
Total files: 45
Passed: 44
Failed: 1
```

### Help and Options

Display command-line help:

```bash
dotnet run --project tools/YamlValidator/ -- --help
```

**Available Options:**
- `-h, --help` - Show help message
- `--schema <path>` - Path to JSON schema file (required)
- `--yaml <path>` - Path to YAML file to validate (mutually exclusive with --validate-all)
- `--validate-all <dir>` - Validate all YAML files in directory recursively (mutually exclusive with --yaml)

## Exit Codes

YamlValidator uses standard exit codes for CI/CD integration:

| Exit Code | Meaning | Description |
|:---------:|---------|-------------|
| `0` | Success | All validated files conform to schema |
| `1` | Failure | Validation errors found or execution error |

**CI Usage:**
```bash
# Build pipeline fails if validation fails
dotnet run --project tools/YamlValidator/ -- \
  --schema tools/YamlSchemaGen/dialogue-schema.json \
  --validate-all Game/YAML/Kojo/ || exit 1
```

## CI Integration

### Git Pre-Commit Hook

Validate staged YAML files before commit (`.githooks/pre-commit`):

```bash
#!/bin/bash

# Get staged YAML files
STAGED_YAML=$(git diff --cached --name-only --diff-filter=ACM | grep '\.yaml$')

if [ -n "$STAGED_YAML" ]; then
  echo "Validating staged YAML files..."

  # Generate latest schema
  dotnet run --project tools/YamlSchemaGen/ || exit 1

  # Validate each staged file
  for file in $STAGED_YAML; do
    dotnet run --project tools/YamlValidator/ -- \
      --schema tools/YamlSchemaGen/dialogue-schema.json \
      --yaml "$file" || exit 1
  done

  echo "YAML validation passed"
fi
```

### GitHub Actions Workflow

Example CI pipeline validation (`.github/workflows/validate-yaml.yml`):

```yaml
name: Validate YAML Dialogue Files

on:
  pull_request:
    paths:
      - 'Game/YAML/Kojo/**/*.yaml'
      - 'tools/YamlSchemaGen/**'
      - 'tools/YamlValidator/**'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup .NET
        uses: actions/setup-dotnet@v3
        with:
          dotnet-version: '8.0.x'

      - name: Generate YAML Schema
        run: dotnet run --project tools/YamlSchemaGen/

      - name: Validate YAML Files
        run: |
          dotnet run --project tools/YamlValidator/ -- \
            --schema tools/YamlSchemaGen/dialogue-schema.json \
            --validate-all Game/YAML/Kojo/
```

### Build Integration

Integrate validation into development build process:

```bash
# Validate before running tests
dotnet run --project tools/YamlSchemaGen/ && \
  dotnet run --project tools/YamlValidator/ -- \
    --schema tools/YamlSchemaGen/dialogue-schema.json \
    --validate-all Game/YAML/Kojo/ && \
  dotnet test
```

## Error Reporting

### Error Message Format

YamlValidator reports errors in a structured format:

```
FAIL: {filename}
Error at {json-path}: {error-kind} - {property-name}
```

**Components:**
- `{filename}` - Name of the failing YAML file
- `{json-path}` - JSONPath to the error location (e.g., `branches[0].condition.TALENT`)
- `{error-kind}` - Type of validation error (e.g., Required, Type, MinItems)
- `{property-name}` - Property that failed validation

### Common Error Types

| Error Kind | Description | Example |
|------------|-------------|---------|
| `Required` | Missing required property | `Error at branches[0]: Required property 'lines' is missing` |
| `Type` | Incorrect data type | `Error at character: Type mismatch - expected string, got integer` |
| `MinItems` | Array too short | `Error at branches[0].lines: Array must contain at least 1 item` |
| `AdditionalProperties` | Unknown property | `Error at branches[0].condition: Additional property 'INVALID' not allowed` |

### YAML Syntax Errors

Parser errors report line numbers:

```
FAIL: COM_K1_0.yaml
Error at line 15: Mapping values are not allowed in this context
```

### Troubleshooting

**Issue: Schema file not found**
```
Error: Could not find file 'dialogue-schema.json'
```
**Solution:** Run YamlSchemaGen first:
```bash
dotnet run --project tools/YamlSchemaGen/
```

**Issue: YAML file not found**
```
FAIL: COM_K1_0.yaml
Error: File not found
```
**Solution:** Verify file path is correct (relative to repository root).

**Issue: All files fail validation after schema update**
```
FAIL: COM_K1_0.yaml
Error at branches[0].condition: Required property 'new_field' is missing
```
**Solution:** Schema breaking change detected. Review schema diff and update YAML files:
```bash
git diff tools/YamlSchemaGen/dialogue-schema.json
```

**Issue: Validation error with no clear location**
```
FAIL: COM_K1_0.yaml
Error at unknown: InvalidType - character
```
**Solution:** Check root-level properties (character, situation, branches) for type mismatches.

## Integration with YamlSchemaGen

YamlValidator requires a schema file generated by YamlSchemaGen:

```bash
# 1. Generate schema (do this once or when schema changes)
dotnet run --project tools/YamlSchemaGen/

# 2. Validate YAML files
dotnet run --project tools/YamlValidator/ -- \
  --schema tools/YamlSchemaGen/dialogue-schema.json \
  --yaml Game/YAML/Kojo/COM_K1_0.yaml
```

For schema generation details and schema versioning, see [YamlSchemaGen README](../YamlSchemaGen/README.md).
