### Key Design Decisions

1. **Delayed Character Injection**: Characters (NPCs) aren't loaded until gameplay progresses. ScenarioParser stores pending injections and HeadlessWindow.Update() calls TryApplyPendingCharacters() periodically.

2. **Internal Types**: VariableResolver and StateInjector are `internal` because they use internal types (VariableCode, CharacterData).

3. **JSON Format**: Simple structure with `name`, `variables` (FLAG:index=value), and `characters` dict.

### Usage Example

```bash
# Create scenario file
echo '{"variables":{"FLAG:0":999},"characters":{"あなた":{"CFLAG:0":1000}}}' > scenario.json

# Run with injection
echo "0\n0\n0\n0\n0\n1000" | dotnet uEmuera.Headless.dll --inject scenario.json Game/
```
