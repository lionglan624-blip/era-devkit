### 描画パイプライン分析

```
スプライト読み込み:
  resources/*.csv → AppContents.LoadContents() → imageDictionary
  ↓
  SpriteManager.cs (Unity) → Texture2D → Sprite

描画フロー (問題あり):
  ERB Command → EmueraConsole.Print* → ConsoleImagePart
  ↓
  ConsoleImagePart.DrawTo() ← コメントアウト！
  GraphicsImage.GDraw*()    ← コメントアウト！
```

### 重要な発見

1. **コマンド定義は完備**: 全ての画像系コマンドが定義・登録されている
2. **状態管理は動作**: GCREATE, GCREATED, GDISPOSE など状態系は正常動作
3. **描画コードは無効**: GraphicsImage.cs の描画メソッド (GDrawCImg, GDrawG等) は全てコメントアウト
4. **ConsoleImagePart.DrawTo() も無効**: 実際の画像レンダリングコードがない
5. **SpriteManager.cs は完全実装**: Unity側のテクスチャ管理は動作する

### 結論

uEmuera は画像系コマンドの**インターフェース**は Emuera から移植済みだが、**実際の描画処理**は実装されていない。これは以下の理由と考えられる：

- WinForms (GDI+) → Unity への描画移植が未完了
- テスト/デバッグ用にコメントアウトされたまま

## Feature 050 実装方針

### 判定結果: **方針B (限定的ERB + Unity補完)**

| 方針 | 条件 | 該当 |
|------|------|------|
| A: ERBライブラリ方式 | G系命令が動作 | × |
| **B: 限定的ERB + Unity補完** | 一部のみ動作 | **○** |
| C: Unity拡張方式 | 非対応 | × |

### 方針B の詳細

1. **利用可能な既存機能**:
   - CBG命令 (データ構造)
   - スプライト状態管理
   - resources CSVからの画像ロード

2. **Unity側で実装が必要**:
   - CBG描画パイプライン (EmueraConsole.CBG.cs のデータを Unity で描画)
   - ConsoleImagePart の Unity 描画

3. **実装優先順位**:
   1. CBGSETSPRITE → Unity描画 (最も実用的)
   2. resources/画像定義 CSV の準備
   3. 必要に応じて G系命令の描画実装
